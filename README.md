# 🤖 ИИ-ассистент онбординга тренеров

Telegram-бот для автоматизации онбординга (адаптации) новых тренеров
АНО ДПО ДО «Байкальский центр спортивного программирования».

Реализация учебного проекта по техническому заданию **ВЯ.ПО.001-01**
(см. отчёт по практике). Бот сопровождает нового тренера по этапам адаптации,
выдаёт учебные материалы из базы знаний, ведёт чек-листы, проводит тестирование
и отвечает на произвольные вопросы с помощью ИИ.

## Возможности

- 📚 **База знаний** — учебные материалы в формате Markdown.
- 🎯 **Этапы онбординга** — последовательное прохождение с сохранением прогресса.
- ✅ **Чек-листы** — интерактивные списки задач по каждому этапу.
- 📝 **Тестирование** — проверка знаний с подсчётом результата.
- ❓ **Вопросы к ИИ** — свободные вопросы, ответ формируется ИИ-ассистентом
  по содержимому базы знаний (RAG).
- 📊 **Прогресс** — отслеживание выполнения этапов и результатов тестов.

## Архитектура

Соответствует подсистемам из ТЗ (раздел 4.1):

| Подсистема (ТЗ)        | Модуль              | Технология                          |
|------------------------|---------------------|-------------------------------------|
| Telegram-интерфейс     | `bot.py`            | aiogram (Telegram Bot API)          |
| ИИ-ассистент           | `ai_agent.py`       | OpenRouter (`openrouter/owl-alpha`) |
| База знаний            | `knowledge_base.py` | Markdown-файлы + лексический поиск   |
| Хранение данных        | `database.py`       | SQLite                              |

> **Примечание о соответствии ТЗ.** Интерфейс реализован на библиотеке **aiogram**,
> работающей напрямую через **Telegram Bot API** — именно так интерфейс указан в
> ТЗ (раздел «Требования к исходным кодам»). Для запуска нужен только токен бота
> от @BotFather, регистрация приложения на my.telegram.org (api_id/api_hash) НЕ
> требуется.
>
> Для подсистемы хранения в ТЗ указана SurrealDB, а для ИИ-ассистента — Claude
> Agent SDK. В настоящем учебном демонстрационном варианте те же функции
> реализованы на более лёгком стеке: ИИ-ассистент построен как агент (системная
> инструкция + поиск по базе знаний + история диалога) на модели
> `openrouter/owl-alpha` через OpenRouter, а хранилище — на встраиваемой СУБД
> SQLite. Функциональные требования раздела 3.1 при этом сохранены.

## Установка и запуск

### 1. Зависимости

```bash
pip install -r requirements.txt
```

### 2. Конфигурация

Скопируйте `.env.example` в `.env` и заполните значения:

```bash
cp .env.example .env
```

- `BOT_TOKEN` — токен бота от [@BotFather](https://t.me/BotFather).
  Это **единственное**, что нужно для Telegram — api_id/api_hash не требуются.
- `OPENROUTER_API_KEY` — ключ с [openrouter.ai](https://openrouter.ai/keys).
- `OPENROUTER_MODEL` — по умолчанию `openrouter/owl-alpha`.
- `TELEGRAM_PROXY` — *необязательно*. Если Telegram заблокирован в вашей сети,
  укажите прокси, например `socks5://user:pass@host:port`. Оставьте пустым, если
  прокси не нужен.

### 3. Запуск

```bash
python bot.py
```

После запуска откройте бота в Telegram и отправьте `/start`.

> ⚠️ **Если Telegram заблокирован.** Чтобы бот достучался до серверов Telegram,
> на машине запуска нужен доступ к ним: укажите прокси в `TELEGRAM_PROXY` либо
> запускайте бота при активном VPN.

## Как сделать скриншоты работы (для отчёта)

1. Запустите бота: `python bot.py`.
2. В Telegram найдите своего бота и отправьте `/start`.
3. Сделайте скриншоты экранов: главное меню, база знаний, этап онбординга с
   чек-листом, прохождение теста, ответ ИИ на свободный вопрос.
4. Для скриншотов на ПК удобно использовать Telegram Desktop.

Готовые примеры экранов лежат в папке `screenshots/`.

## Безопасность

- Все токены и ключи хранятся **только** в `.env` и не попадают в репозиторий
  (`.env` добавлен в `.gitignore`) — соответствует требованию ТЗ 4.6.
- Взаимодействие с внешними сервисами — по HTTPS.
- Персональные данные (Telegram ID, имя) хранятся локально в SQLite.

## Обновление базы знаний

База знаний обновляется без остановки бота: достаточно отредактировать или
добавить `.md`-файл в папку `knowledge_base/` — изменения подхватываются при
следующем обращении (соответствует требованию ТЗ 4.9).
